# Decision Matrix

Use this matrix after parsing reviewer comments.

| Label | Use When | Execution Rule |
| --- | --- | --- |
| `accept` | Request is evidence-backed, in scope, and improves manuscript quality with manageable cost | Execute automatically |
| `partial-accept` | Request is directionally valid but needs narrowed scope or limited wording | Require author confirmation |
| `defer` | Request is low-priority, weakly supported, or high-cost/low-benefit | Require author confirmation |
| `needs-author-confirmation` | Request is ambiguous, conflicting, or impacts core claims/structure | Require author confirmation |

## Aggressive Proposal Trigger Rules

Enter aggressive proposal gate only when one or more conditions hold:
- High-severity reviewer request cannot be satisfied under moderate edits.
- Manuscript logic requires section-level restructuring to remain coherent.
- Reviewer request implies new experiments, new evidence, or significant reframing.

## Minimum Necessary Edit Boundaries

- Reject edits that only improve style but do not answer a reviewer concern.
- Reject edits that expand claims beyond available evidence.
- Prefer local paragraph/sentence edits over structural changes.

## Non-Negotiable Safety Rule

Do not fabricate data, experiments, or citations.
